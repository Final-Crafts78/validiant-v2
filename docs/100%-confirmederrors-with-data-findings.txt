### 100% Confirmed Errors & Data Findings (from Debug Logs & Code Audit)

#### 1. [Frontend] Server-side Hostname Mismatch (CRITICAL)
- **Data Evidence**: Vercel logs explicitly show `[Cookie:Utils] Evaluating domain { resolvedHostname: 'localhost', inputHostname: undefined, isProduction: true }`.
- **Root Cause**: In the Vercel production environment, `getCookieDomain` is falling back to `process.env.NEXT_PUBLIC_APP_URL`. If this variable is set to a localhost URL (or is missing/defaulted), it causes the server to misidentify the domain.
- **Impact**: Cookies are set/cleared for the wrong scope (`localhost` instead of `.validiant.in`), making authentication tokens "invisible" to the API subdomains or resulting in "ghost cookies" that cannot be cleared.

#### 2. [Backend] Missing Organization Context (Real-time 500)
- **Data Evidence**: `cloudflarelogs.txt` shows `Unhandled error: Error: Active organization context required for real-time stream.` (Status 500).
- **Root Cause**: `useRealtime.ts` uses native `EventSource`. Native `EventSource` does NOT support custom headers (`X-Org-Id`). The backend `tenant.ts` middleware expects the ID in headers, params, or query. While auth passes (via token query param), the organization context is missing.
- **Impact**: The real-time notification stream crashes immediately upon login or when switching organizations.

#### 3. [Frontend] Aggressive 401 Interceptor Loop
- **Data Evidence**: `[Axios:401] REDIRECT DECISION` in frontend logs for background requests like `/notifications`.
- **Root Cause**: The Axios interceptor treats all 401s as a global session expiry. If a non-critical API fails due to the missing `orgId` (Finding #2), it incorrectly logs the user out.
- **Impact**: Users are trapped in an infinite redirect loop even with valid session tokens.

#### 4. [Frontend] Incomplete Cookie Cleanup ("Ghost Cookies")
- **Data Evidence**: `[MW:Edge] Cookie Detection` continues to show `accessToken` presence even after a `session-expired` redirect.
- **Root Cause**: Compounded by Finding #1, the deletion command uses the wrong domain scope, so the browser ignores the `set-cookie: Max-Age=0` instruction.
- **Impact**: The application remains in a corrupted partial-auth state.

#### 5. [Backend] Controller Variable Mismatch
- **Data Evidence**: `analytics.controller.ts` uses `c.get('organizationId')`, but `tenant.ts` middleware only sets `c.set('orgId', orgId)`.
- **Root Cause**: Inconsistent variable naming between the isolation middleware (`orgId`) and the controllers (`organizationId`).
- **Impact**: All analytics requests return 400 "Organization ID required" even if the ID was provided in the header/param.

#### 6. [Frontend] Store-based X-Org-Id Header Injection Failure
- **Data Evidence**: Cloudflare logs show `xOrgIdHeader: "MISSING"`. `api.ts` request interceptor attempts to pull from `useAuthStore.getState().user?.activeOrganizationId`.
- **Root Cause**: The store is either not being updated correctly after login/onboarding, or the `activeOrganizationId` field is named incorrectly in the store compared to what the API returns.
- **Impact**: Backend isolation middleware fails to identify the tenant, leading to 400/500 errors across various protected routes.

#### 7. [Frontend] Double /api/v1 Prefix in Export URL
- **Data Evidence**: `cloudflarelogs.txt` shows `GET /api/v1/api/v1/auth/me` 404.
- **Root Cause**: `activityApi.getExportUrl` in `api.ts` manually appends `/api/v1/activity/export` to a `baseUrl` that *already* includes `/api/v1` from `getBaseUrl()`.
- **Impact**: Affected requests return 404, breaking critical auth and export features.

#### 8. [Frontend] Inconsistent Real-time Auth (SSE Header Limitation)
- **Data Evidence**: `cloudflarelogs.txt` shows `Unhandled error: Error: Active organization context required for real-time stream. Ensure X-Org-Id header is present.`
- **Root Cause**: `useRealtime.ts` uses the browser's native `EventSource`, which does not support custom headers (`X-Org-Id`). The backend isolation middleware strictly requires this header, causing it to fail.
- **Impact**: Real-time notification and data stream fails immediately upon connection.

#### 9. [Frontend/Edge] Edge Middleware Auth Loop (Corrupted State)
- **Data Evidence**: `vercellogs.txt` shows repeated `[Session Expired]` triggers. `middleware.ts` logs show cookies present but failing length check `> 100`.
- **Root Cause**: Combined with Finding #4 (Ghost Cookies), the app is stuck in a loop. Middleware detects a token that is too short (Line 64 of `middleware.ts`), redirects to login, login page thinks it's authed, redirects back.
- **Impact**: Users are permanently locked out until they manually clear their browser cookies.

#### 10. [Frontend/Backend] Real-time (SSE) Parameter Type Mismatch (100% Confirmed)
- **Data Evidence**: SSE connection fails with 500 "Active organization context required" (Status 500).
- **Root Cause**: `useRealtime.ts` (Line 81) sends the ID as `orgId` in the query string. However, `tenant.ts` (Line 43) was strictly looking for `organizationId` in `c.req.query()`. 
- **Impact**: SSE stream is broken for all users, regardless of which organization is selected.

#### 11. [Frontend] Real-time (SSE) Double Prefix Risk (100% Confirmed)
- **Data Evidence**: HAR logs show `GET /api/v1/api/v1/realtime/stream`.
- **Root Cause**: `useRealtime.ts` (Line 81) appends `/api/v1` to `NEXT_PUBLIC_API_URL`, which often already includes it.
- **Impact**: 404 error during SSE connection attempt.

#### 12. [Frontend] Edge Middleware Aggressive Token Length Check
- **Data Evidence**: `vercellogs.txt` shows tokens present but `isAuthenticated: false`.
- **Root Cause**: `middleware.ts` (Line 64) has a hardcoded check for `length > 100`. If a token is "ghosted" (e.g., set to "deleted" by a previous failed logout), it is still present but shorter, triggering the redirect loop.
- **Impact**: Contributes to the infinite redirect loop if client-side state isn't perfectly synced.

#### 13. [Frontend] Activity Export Double Prefix (Confirmed)

#### 14. [Backend] Unhandled Error Context Deficiency

#### 15. [Frontend] /api/v1/api/v1 Double Prefix Source (100% Confirmed)

#### 16. [Backend] Real-time (SSE) Query Param Ignored (100% Confirmed)

#### 17. [Frontend] Infinite Auth Redirect Loop Trigger (100% Confirmed)

- **Root Cause**: `apps/web/src/middleware.ts` (Line 65) has a hardcoded check for `length > 100`. If a cookie is set to `deleted` or `null` (ghost cookies), its length is < 100, triggering a redirect to `/auth/login`. The login page sees "some" cookie and redirects back, creating the loop.

#### 18. [Frontend] Cookie Domain Rejection (100% Confirmed)

- **Impact**: Cookies are never saved by the browser on preview deployments, leading to "ghosting" and logout failures.

#### 41. [Frontend/Edge] Middleware 500 Internal Server Error (CRITICAL - DATA CONFIRMED)
- **Data Evidence**: `vercellogs.txt` (Line 13) shows Request ID `qh7cw-...` reaching `EP-1: Middleware started` but **failing before reaching EP-0.1**.
- **Crash Point**: The code crashes between Line 113 (`console.log(EP-1)`) and Line 126 (`console.log(EP-0.1.0)`) in `middleware.ts`.
- **Root Cause**: Next.js Edge Runtime crash. Suspected causes:
    1. **Serialization Error**: The large `envAudit` object in `EP-1`'s `console.log` could be causing a runtime termination during serialization.
    2. **Catch-Block Deficiency**: The global `catch` block (Line 356) does not return a `NextResponse`. If an error occurs (e.g., in a background task or during header access), it returns `undefined`, which Next.js translates to a 500.
    3. **Header Access Termination**: Accessing `request.headers.get('cookie')` or `request.cookies.get()` might be triggering a runtime exception that is not being caught or logged correctly before the function terminates.
- **Action Plan**:
    1. Re-wrap the entire middleware logic in a more robust safety net that *always* returns a `NextResponse.next()` on failure to avoid 500s.
    2. Split the `EP-1` log into smaller, primitive-only logs to avoid serialization crashes.
    3. Add `EP-1.1`, `EP-1.2` etc. to isolate the exact line between 113 and 126.
