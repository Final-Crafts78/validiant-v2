### 100% Confirmed Errors & Data Findings (from Debug Logs)

#### 1. [Frontend] Server-side Hostname Mismatch (CRITICAL)
- **Data Evidence**: Vercel logs explicitly show `[Cookie:Utils] Evaluating domain { resolvedHostname: 'localhost', inputHostname: undefined, isProduction: true }`.
- **Root Cause**: In the Vercel production environment, `getCookieDomain` is falling back to `process.env.NEXT_PUBLIC_APP_URL`. If this variable is set to a localhost URL (or is missing/defaulted), it causes the server to misidentify the domain.
- **Impact**: Cookies are set/cleared for the wrong scope (`localhost` instead of `.validiant.in`), making authentication tokens "invisible" to the API subdomains or resulting in "ghost cookies" that cannot be cleared.

#### 2. [Backend] Missing Organization Context (Real-time 500)
- **Data Evidence**: `cloudflarelogs.txt` shows `Unhandled error: Error: Active organization context required for real-time stream. Ensure X-Org-Id header is present.` (Status 500).
- **Root Cause**: The frontend is failing to consistently pass the `X-Org-Id` header during the transition from onboarding to the dashboard, or the backend is not receiving it.
- **Impact**: The real-time notification stream crashes immediately upon login/onboarding completion.

#### 3. [Frontend] Aggressive 401 Interceptor Loop
- **Data Evidence**: `[Axios:401] REDIRECT DECISION` in frontend logs for background requests like `/notifications`.
- **Root Cause**: The Axios interceptor treats all 401s as a global session expiry. If a non-critical API fails due to the missing `orgId` (Finding #2), it incorrectly logs the user out.
- **Impact**: Users are trapped in an infinite redirect loop even with valid session tokens.

#### 4. [Frontend] Incomplete Cookie Cleanup ("Ghost Cookies")
- **Data Evidence**: `[MW:Edge] Cookie Detection` continues to show `accessToken` presence even after a `session-expired` redirect.
- **Root Cause**: Compounded by Finding #1, the deletion command uses the wrong domain scope, so the browser ignores the `set-cookie: Max-Age=0` instruction.
- **Impact**: The application remains in a corrupted partial-auth state.
