# VALIDIANT — 10-PHASE MASTER BUILD DOCUMENT

## Pre-Phase: Definitive Source of Truth

Every AI implementing any phase MUST read this section first.

### Three Source Repos → One Build Target

| Repo | What to extract |
|---|---|
| `Final-Crafts78/validiant-tracker` (`index.js`) | 15 features — all task/user/upload/export logic |
| `Final-Crafts78/Validiant` (`server.js`) | 10 additional unique features listed below |
| `Final-Crafts78/validiant-v2` (`apps/api/`) | Stack, architecture patterns, all existing TypeScript structure |
| **`Final-Crafts78/ValidiantOld-Newpatchwork`** | **BUILD TARGET — all code goes here** |

### Unique Features Extracted from `Validiant/server.js` — MUST Migrate

These exist in `Validiant` but NOT in `validiant-tracker` and must all be implemented: 
1. **Email notification system** — `sendEmail()` helper (replace `nodemailer` with Resend HTTP API, `nodemailer` is Node.js-only and incompatible with CF Workers)
2. **Contact form** — `POST /api/contact`, inserts into `contact_messages` table
3. **Admin password reset** — `POST /api/users/:id/reset-password` — admin-triggered, returns `tempPassword`
4. **Full employee profile edit** — `PUT /api/users/:id` — name, email, employeeId, phone, optional password
5. **Hard delete user with confirmation** — `DELETE /api/users/:id` — requires `adminPassword` in body, re-verifies admin credentials before deleting, cascades tasks to `Unassigned`
6. **Dedicated status endpoint** — `PUT /api/tasks/:taskId/status` — status-only update, separate from full task update
7. **`address` field** — tasks have a text `address` field separate from `mapUrl`
8. **User-Agent detection in login logs** — detects Mobile vs Web Browser from `User-Agent` header, stores in activity log detail
9. **Parallel analytics** — `Promise.all()` pattern for concurrent DB queries in analytics endpoint
10. **SLA in CSV export** — 72-hour SLA status column (`On Time` / `Overdue` / `In Progress` / `N/A`)

### Runtime Constraints — CF Workers (Non-negotiable)
These Node.js APIs **do not exist** in Cloudflare Workers and must NEVER be used in `apps/api/`:

| ❌ Node-only (forbidden) | ✅ CF Workers replacement |
|---|---|
| `crypto.randomBytes()` | `crypto.getRandomValues(new Uint8Array(n))` |
| `crypto.timingSafeEqual()` | Manual XOR loop (see Phase 3) |
| `bcrypt.hash()` | `crypto.subtle` PBKDF2 (see Phase 3) |
| `multer` | `c.req.formData()` — built into Hono/CF |
| `nodemailer` | Resend HTTP API (free: 3000 emails/mo) |
| `fs`, `path` | Not available — no file system |
| `setInterval` / keep-alive | CF Cron Triggers in `wrangler.toml` |
| `process.env.X` | `c.env.X` via Hono context bindings |
| `sequelize.sync()` | `drizzle-kit push` or `drizzle-kit migrate` |

## Phase 2 — Drizzle Schema & Database Migrations

### Objective
Define every database table, enum, index, and relation using Drizzle ORM. Write all tables in one file (`apps/api/src/db/schema.ts`). Run `drizzle-kit push` to apply to Neon. No application logic, no routes, no controllers.

### File: `apps/api/src/db/index.ts`
```typescript
import { neon } from '@neondatabase/serverless'
import { drizzle } from 'drizzle-orm/neon-http'
import * as schema from './schema'

// Called once per Worker invocation — Neon HTTP is stateless (perfect for CF Workers)
export function getDb(databaseUrl: string) {
  const sql = neon(databaseUrl)
  return drizzle(sql, { schema })
}

export type Database = ReturnType<typeof getDb>
```

### File: `apps/api/src/db/schema.ts` — Complete, exact Drizzle schema
```typescript
import {
  pgTable, pgEnum, serial, text, varchar, boolean,
  integer, real, timestamp, jsonb, index, uniqueIndex
} from 'drizzle-orm/pg-core'
import { relations } from 'drizzle-orm'
import type { Permissions } from '@validiant/shared'

// ─── ENUMS (native PostgreSQL — DB-level enforcement) ───────────────────────

export const systemRoleEnum = pgEnum('system_role', ['admin', 'member', 'guest', 'viewer'])
export const personaEnum = pgEnum('persona', ['project_manager', 'contributor', 'remote_worker', 'habit_tracker'])
export const taskStatusEnum = pgEnum('task_status', [
  'Unassigned', 'Pending', 'Completed', 'Verified',
  'Left Job', 'Not Sharing Info', 'Not Picking',
  'Switch Off', 'Incorrect Number', 'Wrong Address'
])

// ─── USERS TABLE ─────────────────────────────────────────────────────────────

export const users = pgTable('users', {
  id:                 serial('id').primaryKey(),
  name:               varchar('name', { length: 100 }).notNull(),
  email:              varchar('email', { length: 255 }).notNull().unique(),
  passwordHash:       text('password_hash').notNull(),
  // Format: "pbkdf2:sha512:600000:{salt_base64url}:{hash_base64url}"
  systemRole:         systemRoleEnum('system_role').notNull().default('member'),
  persona:            personaEnum('persona').notNull().default('contributor'),
  permissions:        jsonb('permissions').$type<Permissions>().notNull(),
  // permissions is always set from DEFAULT_PERMISSIONS[systemRole] on creation,
  // but can be overridden per-user by admin (Option C RBAC)
  employeeId:         varchar('employee_id', { length: 50 }).unique(),
  phone:              varchar('phone', { length: 30 }),
  isActive:           boolean('is_active').notNull().default(true),
  mustChangePassword: boolean('must_change_password').notNull().default(true),
  loginAttempts:      integer('login_attempts').notNull().default(0),
  lockoutUntil:       timestamp('lockout_until', { withTimezone: true }),
  passwordChangedAt:  timestamp('password_changed_at', { withTimezone: true }),
  lastActiveAt:       timestamp('last_active_at', { withTimezone: true }),
  // System user placeholder — noted, not implemented (Phase Q4 answer)
  // isSystemUser: boolean('is_system_user').notNull().default(false),
  createdAt:          timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
  updatedAt:          timestamp('updated_at', { withTimezone: true }).notNull().defaultNow(),
}, (t) => ({
  emailIdx:      uniqueIndex('users_email_idx').on(t.email),
  roleIdx:       index('users_role_idx').on(t.systemRole),
  personaIdx:    index('users_persona_idx').on(t.persona),
  employeeIdIdx: uniqueIndex('users_employee_id_idx').on(t.employeeId),
  activeIdx:     index('users_active_idx').on(t.isActive),
}))

// ─── TASKS TABLE ─────────────────────────────────────────────────────────────

export const tasks = pgTable('tasks', {
  id:                   serial('id').primaryKey(),
  title:                varchar('title', { length: 500 }).notNull(),   // Case ID
  clientName:           varchar('client_name', { length: 200 }),
  pincode:              varchar('pincode', { length: 6 }),
  // Constraint: pincode must be exactly 6 digits — enforced at app layer via Zod
  address:              text('address'),                                 // From Validiant repo
  mapUrl:               text('map_url'),
  latitude:             real('latitude'),
  longitude:            real('longitude'),
  assignedTo:           integer('assigned_to').references(() => users.id, { onDelete: 'set null' }),
  assignedDate:         varchar('assigned_date', { length: 10 }),       // "YYYY-MM-DD"
  assignedAtTimestamp:  timestamp('assigned_at_timestamp', { withTimezone: true }),
  status:               taskStatusEnum('status').notNull().default('Unassigned'),
  notes:                text('notes'),
  manualDate:           varchar('manual_date', { length: 10 }),
  manualTime:           varchar('manual_time', { length: 8 }),
  completedAt:          timestamp('completed_at', { withTimezone: true }),
  verifiedAt:           timestamp('verified_at', { withTimezone: true }),
  createdBy:            integer('created_by').references(() => users.id, { onDelete: 'set null' }),
  createdAt:            timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
  updatedAt:            timestamp('updated_at', { withTimezone: true }).notNull().defaultNow(),
}, (t) => ({
  assignedToIdx:   index('tasks_assigned_to_idx').on(t.assignedTo),
  statusIdx:       index('tasks_status_idx').on(t.status),
  pincodeIdx:      index('tasks_pincode_idx').on(t.pincode),
  assignedDateIdx: index('tasks_assigned_date_idx').on(t.assignedDate),
  createdAtIdx:    index('tasks_created_at_idx').on(t.createdAt),
}))

// ─── ACTIVITY LOGS TABLE ─────────────────────────────────────────────────────

export const activityLogs = pgTable('activity_logs', {
  id:         serial('id').primaryKey(),
  userId:     integer('user_id').references(() => users.id, { onDelete: 'set null' }),
  userName:   varchar('user_name', { length: 100 }),
  action:     varchar('action', { length: 100 }).notNull(),
  // Action values: LOGIN_SUCCESS, LOGIN_FAILED, TASK_CREATED, TASK_UPDATED,
  // TASK_ASSIGNED, TASK_UNASSIGNED, TASK_REASSIGNED, TASK_DELETED,
  // TASK_STATUS_CHANGED, BULK_UPLOAD_COMPLETED, USER_CREATED,
  // USER_UPDATED, USER_DELETED, PASSWORD_RESET, EMPLOYEE_CREATED
  taskId:     integer('task_id'),
  taskTitle:  varchar('task_title', { length: 500 }),
  oldValue:   jsonb('old_value'),
  newValue:   jsonb('new_value'),
  details:    text('details'),         // Human-readable detail string
  deviceType: varchar('device_type', { length: 20 }), // "Mobile App" | "Web Browser" | "Unknown"
  ipAddress:  varchar('ip_address', { length: 50 }),
  timestamp:  timestamp('timestamp', { withTimezone: true }).notNull().defaultNow(),
}, (t) => ({
  userIdIdx:   index('logs_user_id_idx').on(t.userId),
  taskIdIdx:   index('logs_task_id_idx').on(t.taskId),
  actionIdx:   index('logs_action_idx').on(t.action),
  timestampIdx:index('logs_timestamp_idx').on(t.timestamp),
}))

// ─── CONTACT MESSAGES TABLE (from Validiant repo) ────────────────────────────

export const contactMessages = pgTable('contact_messages', {
  id:        serial('id').primaryKey(),
  name:      varchar('name', { length: 100 }).notNull(),
  email:     varchar('email', { length: 255 }).notNull(),
  message:   text('message').notNull(),
  status:    varchar('status', { length: 20 }).notNull().default('new'),
  // status values: "new" | "read" | "replied"
  createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
})

// ─── RELATIONS ───────────────────────────────────────────────────────────────

export const usersRelations = relations(users, ({ many }) => ({
  tasks:        many(tasks,        { relationName: 'assignedTasks' }),
  createdTasks: many(tasks,        { relationName: 'createdTasks' }),
  activityLogs: many(activityLogs, { relationName: 'userLogs' }),
}))

export const tasksRelations = relations(tasks, ({ one }) => ({
  assignedUser: one(users, {
    fields: [tasks.assignedTo],
    references: [users.id],
    relationName: 'assignedTasks',
  }),
  createdByUser: one(users, {
    fields: [tasks.createdBy],
    references: [users.id],
    relationName: 'createdTasks',
  }),
}))
```

## Phase 3 — Above-Elite Auth System (PBKDF2 + JWT + Redis Lockout)

### Objective
Implement the complete authentication system. All crypto uses **Web Crypto API only** — no `bcryptjs`, no Node `crypto` module. This is the security foundation every other phase depends on.

### Why PBKDF2 over bcrypt on CF Workers
CF Workers' Web Crypto API offloads PBKDF2 to **hardware-accelerated native crypto** — it runs outside the JavaScript CPU time budget entirely. bcrypt is pure JavaScript and would fail the 10–30ms CPU limit. PBKDF2-SHA-512 with 600,000 iterations is the NIST SP 800-132 (2023) recommendation and OWASP 2024 minimum.

```typescript
// Web Crypto API — available natively in CF Workers runtime
// NO imports needed. Uses global `crypto` object.

const PBKDF2_ITERATIONS = 600_000   // NIST SP 800-132 (2023) recommendation for SHA-512
const PBKDF2_HASH = 'SHA-512'
const DERIVED_KEY_BITS = 512        // 64 bytes
const SALT_BYTES = 32               // 256-bit salt

export async function hashPassword(password: string): Promise<string> {
  const salt = crypto.getRandomValues(new Uint8Array(SALT_BYTES))
  
  const keyMaterial = await crypto.subtle.importKey(
    'raw',
    new TextEncoder().encode(password),
    { name: 'PBKDF2' },
    false,
    ['deriveBits']
  )

  const derivedBits = await crypto.subtle.deriveBits(
    {
      name: 'PBKDF2',
      salt: salt,
      iterations: PBKDF2_ITERATIONS,
      hash: PBKDF2_HASH,
    },
    keyMaterial,
    DERIVED_KEY_BITS
  )

  const saltB64 = btoa(String.fromCharCode(...salt))
  const hashB64 = btoa(String.fromCharCode(...new Uint8Array(derivedBits)))

  // Storage format: algorithm:hash:iterations:salt:derivedKey
  return `pbkdf2:sha512:${PBKDF2_ITERATIONS}:${saltB64}:${hashB64}`
}

export async function verifyPassword(password: string, stored: string): Promise<boolean> {
  const parts = stored.split(':')
  // Expected: ["pbkdf2", "sha512", "600000", saltB64, hashB64]
  if (parts.length !== 5 || parts[0] !== 'pbkdf2') return false

  const [, , iterStr, saltB64, expectedHashB64] = parts
  const iterations = parseInt(iterStr, 10)
  
  const salt = Uint8Array.from(atob(saltB64), c => c.charCodeAt(0))
  const expectedHash = Uint8Array.from(atob(expectedHashB64), c => c.charCodeAt(0))

  const keyMaterial = await crypto.subtle.importKey(
    'raw',
    new TextEncoder().encode(password),
    { name: 'PBKDF2' },
    false,
    ['deriveBits']
  )

  const derivedBits = await crypto.subtle.deriveBits(
    { name: 'PBKDF2', salt, iterations, hash: 'SHA-512' },
    keyMaterial,
    DERIVED_KEY_BITS
  )

  const actualHash = new Uint8Array(derivedBits)

  // Constant-time comparison — prevents timing attacks
  return constantTimeEqual(actualHash, expectedHash)
}

// Timing-safe equality — always iterates full length regardless of mismatch
function constantTimeEqual(a: Uint8Array, b: Uint8Array): boolean {
  if (a.length !== b.length) return false
  let diff = 0
  for (let i = 0; i < a.length; i++) {
    diff |= a[i] ^ b[i]
  }
  return diff === 0
}

// Generate cryptographically secure random password for new employees
// Always satisfies all complexity requirements
export function generateSecurePassword(): string {
  const upper = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ'
  const lower = 'abcdefghijklmnopqrstuvwxyz'
  const digits = '0123456789'
  const special = '!@#$%^&*()_+-='
  const all = upper + lower + digits + special

  const arr = crypto.getRandomValues(new Uint8Array(16))
  let password = ''
  // Guarantee at least one of each required character class
  password += upper[arr[0] % upper.length]
  password += lower[arr[1] % lower.length]
  password += digits[arr[2] % digits.length]
  password += special[arr[3] % special.length]
  // Fill remaining 12 chars from full charset
  for (let i = 4; i < 16; i++) {
    password += all[arr[i] % all.length]
  }
  // Shuffle using Fisher-Yates with new random values
  const shuffleArr = crypto.getRandomValues(new Uint8Array(16))
  const chars = password.split('')
  for (let i = chars.length - 1; i > 0; i--) {
    const j = shuffleArr[i] % (i + 1)
    ;[chars[i], chars[j]] = [chars[j], chars[i]]
  }
  return chars.join('')
}
```

```typescript
import { SignJWT, jwtVerify } from 'jose'

export type JWTPayload = {
  sub: string       // user.id as string
  role: string      // systemRole
  persona: string
  iat: number
  exp: number
}

export async function signAccessToken(
  payload: Omit<JWTPayload, 'iat' | 'exp'>,
  secret: string
): Promise<string> {
  return new SignJWT(payload)
    .setProtectedHeader({ alg: 'HS256' })
    .setIssuedAt()
    .setExpirationTime('15m')   // 15-minute access token
    .sign(new TextEncoder().encode(secret))
}

export async function signRefreshToken(
  userId: string,
  secret: string
): Promise<string> {
  return new SignJWT({ sub: userId })
    .setProtectedHeader({ alg: 'HS256' })
    .setIssuedAt()
    .setExpirationTime('7d')    // 7-day refresh token
    .sign(new TextEncoder().encode(secret))
}

export async function verifyToken(token: string, secret: string): Promise<JWTPayload | null> {
  try {
    const { payload } = await jwtVerify(token, new TextEncoder().encode(secret))
    return payload as JWTPayload
  } catch {
    return null
  }
}
```

```typescript
import { Redis } from '@upstash/redis'

export function getRedis(env: { UPSTASH_REDIS_REST_URL: string; UPSTASH_REDIS_REST_TOKEN: string }) {
  return new Redis({
    url: env.UPSTASH_REDIS_REST_URL,
    token: env.UPSTASH_REDIS_REST_TOKEN,
  })
}

// Login lockout: 5 failed attempts → 15-minute lockout, tracked in Redis
const LOCKOUT_MAX_ATTEMPTS = 5
const LOCKOUT_WINDOW_SECONDS = 15 * 60 // 15 minutes
const LOCKOUT_KEY = (email: string) => `lockout:${email}`

export async function checkAndIncrementLoginAttempts(
  redis: Redis,
  email: string
): Promise<{ locked: boolean; remainingSeconds?: number }> {
  const key = LOCKOUT_KEY(email)
  const attempts = await redis.incr(key)
  
  if (attempts === 1) {
    // First attempt — set expiry window
    await redis.expire(key, LOCKOUT_WINDOW_SECONDS)
  }
  
  if (attempts >= LOCKOUT_MAX_ATTEMPTS) {
    const ttl = await redis.ttl(key)
    return { locked: true, remainingSeconds: ttl }
  }
  
  return { locked: false }
}

export async function clearLoginAttempts(redis: Redis, email: string): Promise<void> {
  await redis.del(LOCKOUT_KEY(email))
}

// Refresh token blocklist (for logout invalidation)
export async function blockRefreshToken(redis: Redis, token: string, expirySeconds: number): Promise<void> {
  await redis.setex(`blocklist:${token}`, expirySeconds, '1')
}

export async function isRefreshTokenBlocked(redis: Redis, token: string): Promise<boolean> {
  const val = await redis.get(`blocklist:${token}`)
  return val !== null
}
```

```typescript
import { createMiddleware } from 'hono/factory'
import { verifyToken } from '../lib/jwt'
import type { Env } from '../index'
import type { JWTPayload } from '../lib/jwt'

export type AuthContext = {
  Variables: {
    user: JWTPayload
  }
}

// Attaches verified user payload to c.var.user
// All protected routes MUST use this middleware
export const authMiddleware = createMiddleware<{ Bindings: Env } & AuthContext>(async (c, next) => {
  const authHeader = c.req.header('Authorization')
  if (!authHeader?.startsWith('Bearer ')) {
    return c.json({ success: false, message: 'Missing authorization token' }, 401)
  }

  const token = authHeader.slice(7)
  const payload = await verifyToken(token, c.env.JWT_ACCESS_SECRET)

  if (!payload) {
    return c.json({ success: false, message: 'Invalid or expired token' }, 401)
  }

  c.set('user', payload)
  await next()
})
```

```typescript
import { createMiddleware } from 'hono/factory'

import type { Env } from '../index'

// Checks a specific permission flag from the user's JSONB permissions field
// Usage: app.delete('/api/tasks/:id', authMiddleware, requirePermission('tasks', 'delete'), handler)
export function requirePermission<
  Category extends keyof Permissions,
  Action extends keyof Permissions[Category]
>(category: Category, action: Action) {
  return createMiddleware<{ Bindings: Env } & AuthContext>(async (c, next) => {
    const user = c.var.user
    const db = getDb(c.env.DATABASE_URL)
    
    const { users } = await import('../db/schema')
    const { eq } = await import('drizzle-orm')
    
    const [dbUser] = await db
      .select({ permissions: users.permissions, isActive: users.isActive })
      .from(users)
      .where(eq(users.id, parseInt(user.sub)))
      .limit(1)
    
    if (!dbUser || !dbUser.isActive) {
      return c.json({ success: false, message: 'User not found or inactive' }, 403)
    }
    
    const allowed = dbUser.permissions?.[category]?.[action as string]
    if (!allowed) {
      return c.json({ 
        success: false, 
        message: `Permission denied: ${String(category)}.${String(action)}` 
      }, 403)
    }
    
    await next()
  })
}
```

```typescript
import { Hono } from 'hono'
import { zValidator } from '@hono/zod-validator'
import { z } from 'zod'
import { getDb } from '../db'
import { users } from '../db/schema'
import { eq } from 'drizzle-orm'
import { hashPassword, verifyPassword, generateSecurePassword } from '../lib/crypto'
import { signAccessToken, signRefreshToken, verifyToken } from '../lib/jwt'
import { getRedis, checkAndIncrementLoginAttempts, clearLoginAttempts, blockRefreshToken, isRefreshTokenBlocked } from '../lib/redis'
import { logActivity } from '../services/activityLog'
import type { Env } from '../index'
import { DEFAULT_PERMISSIONS } from '@validiant/shared'

const auth = new Hono<{ Bindings: Env }>()

// ─── POST /api/auth/login ────────────────────────────────────────────────────
auth.post('/login', zValidator('json', z.object({
  email: z.string().email(),
  password: z.string().min(1),
})), async (c) => {
  const { email, password } = c.req.valid('json')
  const db = getDb(c.env.DATABASE_URL)
  const redis = getRedis(c.env)

  // Check Redis lockout BEFORE any DB query (fast path)
  const lockoutKey = `lockout:${email}`
  const currentAttempts = await redis.get<number>(lockoutKey) ?? 0
  if (currentAttempts >= 5) {
    const ttl = await redis.ttl(lockoutKey)
    return c.json({ success: false, message: `Account locked. Try again in ${Math.ceil(ttl / 60)} minutes.` }, 429)
  }

  // Always run crypto.subtle verify even if user not found (timing attack prevention)
  // Use a dummy hash that will always fail but takes same time
  const DUMMY_HASH = 'pbkdf2:sha512:600000:AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA=:AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA='

  const [user] = await db
    .select()
    .from(users)
    .where(eq(users.email, email.toLowerCase()))
    .limit(1)

  const hashToVerify = user?.passwordHash ?? DUMMY_HASH
  const isValid = await verifyPassword(password, hashToVerify)

  if (!user || !isValid || !user.isActive) {
    const lockResult = await checkAndIncrementLoginAttempts(redis, email)
    
    await logActivity(db, null, email, 'LOGIN_FAILED', null, null, null, null, {
      reason: !user ? 'user_not_found' : !isValid ? 'invalid_password' : 'account_inactive',
      deviceType: c.req.header('User-Agent')?.includes('Mobile') ? 'Mobile App' : 'Web Browser',
    })

    if (lockResult.locked) {
      return c.json({ success: false, message: `Account locked for ${Math.ceil((lockResult.remainingSeconds ?? 900) / 60)} minutes.` }, 429)
    }

    return c.json({ success: false, message: 'Invalid email or password' }, 401)
  }

  // Success — clear lockout, update last active
  await clearLoginAttempts(redis, email)
  await db.update(users).set({ lastActiveAt: new Date() }).where(eq(users.id, user.id))

  const accessToken = await signAccessToken(
    { sub: String(user.id), role: user.systemRole, persona: user.persona },
    c.env.JWT_ACCESS_SECRET
  )
  const refreshToken = await signRefreshToken(String(user.id), c.env.JWT_REFRESH_SECRET)

  const deviceType = c.req.header('User-Agent')?.includes('Mobile') ? 'Mobile App' : 'Web Browser'
  await logActivity(db, user.id, user.name, 'LOGIN_SUCCESS', null, null, null, null, {
    deviceType,
  })

  return c.json({
    success: true,
    accessToken,
    refreshToken,
    mustChangePassword: user.mustChangePassword,
    user: { id: user.id, name: user.name, role: user.systemRole, persona: user.persona, email: user.email }
  })
})

// ─── POST /api/auth/refresh ──────────────────────────────────────────────────
auth.post('/refresh', zValidator('json', z.object({ refreshToken: z.string() })), async (c) => {
  const { refreshToken } = c.req.valid('json')
  const redis = getRedis(c.env)

  if (await isRefreshTokenBlocked(redis, refreshToken)) {
    return c.json({ success: false, message: 'Token revoked' }, 401)
  }

  const payload = await verifyToken(refreshToken, c.env.JWT_REFRESH_SECRET)
  if (!payload) return c.json({ success: false, message: 'Invalid refresh token' }, 401)

  const db = getDb(c.env.DATABASE_URL)
  const [user] = await db.select().from(users).where(eq(users.id, parseInt(payload.sub))).limit(1)
  if (!user || !user.isActive) return c.json({ success: false, message: 'User not found' }, 401)

  const newAccessToken = await signAccessToken(
    { sub: String(user.id), role: user.systemRole, persona: user.persona },
    c.env.JWT_ACCESS_SECRET
  )

  return c.json({ success: true, accessToken: newAccessToken })
})

// ─── POST /api/auth/logout ───────────────────────────────────────────────────
auth.post('/logout', zValidator('json', z.object({ refreshToken: z.string() })), async (c) => {
  const { refreshToken } = c.req.valid('json')
  const redis = getRedis(c.env)
  // Block refresh token for 7 days (its max lifetime)
  await blockRefreshToken(redis, refreshToken, 7 * 24 * 60 * 60)
  return c.json({ success: true, message: 'Logged out' })
})

// ─── POST /api/auth/change-password ─────────────────────────────────────────
auth.post('/change-password', zValidator('json', z.object({
  currentPassword: z.string().min(1),
  newPassword: z.string()
    .min(10, 'Minimum 10 characters')
    .regex(/[A-Z]/, 'Must contain uppercase')
    .regex(/[a-z]/, 'Must contain lowercase')
    .regex(/[0-9]/, 'Must contain digit')
    .regex(/[!@#$%^&*()_+\-=\[\]{}|;:,.<>?]/, 'Must contain special character'),
})), async (c) => {
  const authHeader = c.req.header('Authorization')
  if (!authHeader?.startsWith('Bearer ')) return c.json({ success: false, message: 'Unauthorized' }, 401)
  
  const { verifyToken } = await import('../lib/jwt')
  const payload = await verifyToken(authHeader.slice(7), c.env.JWT_ACCESS_SECRET)
  if (!payload) return c.json({ success: false, message: 'Invalid token' }, 401)

  const { currentPassword, newPassword } = c.req.valid('json')
  const db = getDb(c.env.DATABASE_URL)
  
  const [user] = await db.select().from(users).where(eq(users.id, parseInt(payload.sub))).limit(1)
  if (!user) return c.json({ success: false, message: 'User not found' }, 404)

  const isValid = await verifyPassword(currentPassword, user.passwordHash)
  if (!isValid) return c.json({ success: false, message: 'Current password is incorrect' }, 400)

  const newHash = await hashPassword(newPassword)
  await db.update(users).set({
    passwordHash: newHash,
    mustChangePassword: false,
    passwordChangedAt: new Date(),
    updatedAt: new Date(),
  }).where(eq(users.id, user.id))

  return c.json({ success: true, message: 'Password changed successfully' })
})
```

```typescript
// Extracted from validiant-tracker index.js and Validiant server.js — identical logic
export function extractCoordinates(url: string): { latitude: number; longitude: number } | null {
  if (!url) return null

  // Pattern 1: @lat,lng (Google Maps standard)
  const p1 = url.match(/@(-?\d+\.\d+),(-?\d+\.\d+)/)
  if (p1) return { latitude: parseFloat(p1[1]), longitude: parseFloat(p1[2]) }

  // Pattern 2: ?q=lat,lng
  const p2 = url.match(/\?q=(-?\d+\.\d+),(-?\d+\.\d+)/)
  if (p2) return { latitude: parseFloat(p2[1]), longitude: parseFloat(p2[2]) }

  return null
}

// Haversine formula — used in employee task sorting (nearest first)
export function haversineDistance(
  lat1: number, lng1: number,
  lat2: number, lng2: number
): number {
  const R = 6371 // km
  const dLat = (lat2 - lat1) * Math.PI / 180
  const dLng = (lng2 - lng1) * Math.PI / 180
  const a = Math.sin(dLat/2)**2 +
            Math.cos(lat1 * Math.PI/180) * Math.cos(lat2 * Math.PI/180) *
            Math.sin(dLng/2)**2
  return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a))
}
```

**CSV export — SLA calculation logic (from Validiant server.js):**
```typescript
function calculateSLA(task: Task): string {
  if (task.completedAt && task.assignedAtTimestamp) {
    const hours = (new Date(task.completedAt).getTime() - new Date(task.assignedAtTimestamp).getTime()) / 3_600_000
    return hours <= 72 ? 'On Time' : 'Overdue'
  }
  if (task.status === 'Pending' && task.assignedAtTimestamp) {
    const hours = (Date.now() - new Date(task.assignedAtTimestamp).getTime()) / 3_600_000
    return hours <= 72 ? 'In Progress' : 'Overdue'
  }
  return 'N/A'
}
```

### File: `apps/web/src/lib/apiClient.ts`
```typescript
import axios from 'axios'

const API_BASE = import.meta.env.VITE_API_URL ?? 'http://localhost:8787'

export const apiClient = axios.create({
  baseURL: `${API_BASE}/api`,
  headers: { 'Content-Type': 'application/json' },
})

// Request interceptor — attaches access token from sessionStorage
apiClient.interceptors.request.use((config) => {
  const token = sessionStorage.getItem('accessToken')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// Response interceptor — handles 401 by attempting token refresh
// On refresh success: retry original request
// On refresh failure: clear tokens, redirect to /login
apiClient.interceptors.response.use(
  (res) => res,
  async (error) => {
    if (error.response?.status === 401 && !error.config._retry) {
      error.config._retry = true
      const refreshToken = sessionStorage.getItem('refreshToken')
      if (refreshToken) {
        try {
          const { data } = await axios.post(`${API_BASE}/api/auth/refresh`, { refreshToken })
          sessionStorage.setItem('accessToken', data.accessToken)
          error.config.headers.Authorization = `Bearer ${data.accessToken}`
          return apiClient(error.config)
        } catch {
          sessionStorage.clear()
          window.location.href = '/login'
        }
      }
    }
    return Promise.reject(error)
  }
)
```
